Avoid Dependency versions conflicts While running unit tests locally for an OpenStack Project

April 4, 2018 / by Manjeet S. Bhatia

Running unit test for project with modified versioned of dependency can be tricky. OpenStack has requirements project where all the upper constraints for version of each project used are defined. When the unit tests or any tests are fired the first thing done is creating a virtual environment and then installing all the dependencies defined in requirement.txt and test-requirements.txt of the project repo. Version of each dependency is chosen from upper-constraints.txt of requiremnts repo.

To run tests for project with modified dependency requirements.txt and tox.ini of project needs to be modified..

Edit tox.ini

I have to run tests for neutron with modified neutron-lib and networking-odl. In tox.ini there will be install_command = {toxinidir}/tools/tox_install.sh {env:UPPER_CONSTRAINTS_FILE:https://git.openstack.org/cgit/openstack/ requirements/plain/upper-constraints.txt} {opts} {packages} in [testenv].

1. wget https://git.openstack.org/cgit/openstack/requirements/plain/upper-constraints.txt

2. edit replace 'neutron-lib>=some version' with -e file:///opt/stack/new/neutron-lib#egg=neutron-lib

Note: file path could be different based on where you have neutron-lib located locally.

3. Replace UPPER_CONSTRAINTS_FILE:'local_file_path' in tox.ini with new upper-constraints.txt

4. Edit requirements.txt from neutron repo to replace 'neutron-lib>="some version" with -e file:///opt/stack/new/neutron-lib#egg=neutron-lib

Run Tests tox -e py27/pep8/**
